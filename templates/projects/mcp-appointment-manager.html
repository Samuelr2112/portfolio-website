Python FastAPI MCP Appointment Manager (AI Integration)

Intelligent appointment management backend built with FastAPI and PostgreSQL, containerized with Docker and integrated with Claude via Model Context Protocol (MCP). This project demonstrates how artificial intelligence can directly interact with a backend system to create, list, update, and delete appointments.

Technologies Used:
Python FastAPI PostgreSQL Docker Claude AI MCP Protocol
View on GitHub

Key Features

AI-Powered Management

Claude AI can control and query the API/database through natural language, making appointment management intuitive and efficient.

Complete CRUD Operations

Full Create, Read, Update, Delete functionality for appointment management with FastAPI and PostgreSQL.

Containerized Deployment

Docker containerization ensures consistent deployment across different environments and easy scalability.

MCP Integration

Model Context Protocol integration allows seamless communication between Claude AI and the backend system.

Project Origin & Inspiration

I first came across the idea in a Network Chuck YouTube video where he showed Docker integrating with different AI models. Honestly, I was blown away, it made me realize how powerful the Model Context Protocol (MCP) could be, and I felt like I had to try building something of my own.

What really pushed me was the fact that MCP is still so new and there’s almost no information out there. That made it even more exciting to experiment, knowing I’d be working on something at the very edge of what’s possible.

I chose FastAPI because of its speed and simplicity compared to other frameworks like Flask or Django. But more than that, I wanted to start integrating AI into my daily developer workflow. I’m convinced we’re not even at 10% of AI’s real potential yet, so this project was my way of getting ahead and learning by doing.

The Problem

The real-world issue I’ve noticed is that many businesses aren’t always fully transparent about scheduling. In fields like medicine, I’ve personally seen situations where appointments are pushed further than necessary, but when you insist on urgency, suddenly a slot “appears.” That lack of visibility inspired me to explore a more open and AI-driven way to manage logistics.

Even in more straightforward businesses, like my friend’s moving company or Aldea Music Corp, the burden of manually managing appointments is heavy. They may not deal with dishonesty, but they still have to either do the scheduling themselves or hire someone just for that task. That’s inefficient, and I wanted to see if AI could simplify it.

Technical Challenges

AI Integration Complexity

One of the hardest errors I faced was connecting the remote server and Claude with the backend. At one point, I almost thought I wouldn’t get it working. I managed to push through using Claude itself, Network Chuck’s video, and the VS Code extension SFTP from Natizyskunk to debug.

MCP Server Configuration

The most frustrating part was getting the MCP Business Assistant server to run smoothly with Claude without forcing me to enter parameters manually every time. I seriously considered dropping the project at that point, but finding a workaround became one of the most rewarding breakthroughs.

Future Roadmap

User Interface

The biggest missing piece is a UI where businesses can actually interact with the system. Right now it’s API-only, but I want to design a base model and dashboard so companies can integrate their data directly.

External Calendar Integration

Syncing with Google Calendar or Outlook would make the system instantly more practical for real businesses.

AI and Voice Features

I plan to experiment with adding more AI capabilities and even voice control, so users can book or update appointments hands-free.

Subscription Service

In the long term, this could evolve into a subscription-based service for ongoing maintenance and support, making it directly useful to companies.

Project Impact

On the technical side, this project proved to me that lightweight tools like FastAPI and Docker can be combined with cutting-edge AI to create workflows that feel futuristic but are actually achievable today. It also showed me how important it is to design APIs with validation and error handling when you rely on AI to send requests.

On the personal side, it boosted my confidence as a developer. I went from “can I even connect these things?” to “I just built an AI-driven backend that runs in containers and talks to a real database.” That leap alone made the effort worthwhile.

Usage Example

Natural language with Claude:

  • “Show me all existing appointments.”
  • “Edit the appointment for Arturo Martinez and move it to Friday at 10 AM.”
  • “Add a new appointment for Samuel Rincon tomorrow at 3 PM.”
  • “Delete the appointment with ID 7.”

Claude translates these instructions into structured API calls through MCP: 


    GET /appointments
    [
    { "id": 1, "client": "Arturo Martinez", "date": "2025-09-29", "time": "14:00" }
